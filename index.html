✨ これは何？

「○○の教科書を作って」と日本語で話しかけるだけで、AIが：

  1. テーマを 観点3〜5本 に分解する
  2. 並列に4人のAI がリサーチする
  3. 9章構成の教科書 を組み立てる
  4. NotebookLMにアップロード する
  5. マインドマップ・音声・FAQ を自動生成する

…までを一晩で完了させる、Claude Code 用のスキルです。

完成すると、いつでもブラウザ／スマホで勉強できる学習ハブが手に入ります。

🎯 こんな人に向いている

  • 本気で学びたいテーマがある人
  • 自分の言葉で語れるようになりたい人
  • AIに「使われる」のではなく「使う」側に立ちたい人
  • 発信・出版・講座を目指している人

逆に向いていない：「効率よく知識を頭に入れたい」だけの人、メソッドを暗記したいだけの人。 このメソッドの目的は 学びの過程で自分と向き合う時間 を含むので。

🆘 うまくいかなかったら？ 慌てなくて大丈夫。Claude Code に 「うまくいきません。直してください」 と書くだけです。 Python が無い・pip が無い・権限エラー…どんなエラーでも Claude が自動で診断して、解決コマンドを実行してくれます。詳しくは下の 🛠 トラブル対応 セクション。

🛠 うまくいかなかったとき

Claude Code Desktop の最大の強みは、エラーが出たらそのまま Claude に直してと頼めること。普通のターミナルと違い、AI が原因を診断して修正コマンドを実行してくれます。

🪄 万能の呪文

どんなエラーでも、まずこれをチャット欄にコピペして送ってください：

上のインストールがうまくいきませんでした。
エラーを見て、原因を直してから、もう一度インストールを最後まで進めてください。
途中で必要なソフト（PythonやHomebrew等）が足りなければ、それも入れてください。

これだけで、Claude Code が：

  1. エラーログを読む
  2. 原因を特定する
  3. 解決コマンドを提案して実行する（毎回 Yes/Allow をクリックするだけ）
  4. インストールを最後まで通す

📋 よくあるエラーと、Claude がやってくれること

エラー原因Claude が自動でやること
command not found: python3 / Python 3 が見つかりません Python が入っていない Mac: brew install python3
Windows: winget install Python.Python.3
→ 完了後、インストールを再開
command not found: pip pip が無い python3 -m ensurepip --upgrade を実行
command not found: brew Homebrew が無い（Mac） Homebrew 公式インストーラを実行
Permission denied / 権限がありません システム領域に書き込もうとした pip install --user または sudo 付きで再実行（パスワード入力を求められます）
externally-managed-environment 最新のMacで pip 直接実行が制限される --break-system-packages オプション付き or 仮想環境で実行
ブラウザが開かない / Google ログインで詰まる OAuth フロー失敗 notebooklm login を単独で再実行 → 詳細ログから原因特定
NotebookLM の音声生成が失敗 Google のレート制限 5〜10分待ってから再試行（Claude が時間を空けて再呼び出し）

💬 よくある不安への回答

「私はプログラミングできないんですが…」

このガイドの通りに、コピペだけで進められます。料理のレシピを「分子レベルで理解する必要がない」のと同じです。

「途中で失敗したら全部壊れません?」

壊れません。失敗してもパソコンには何の影響もありません。何度でもやり直せます。 しかも Claude Code Desktop なら「直して」と書けば AI が自動で復旧します。

「変な英語が画面に流れて怖い」

英語のメッセージは「いま何をしているか」の実況中継。読まなくて大丈夫。 ✅ と 🎉 マークだけ見ていればOK。

「インストールしたものを後で消したい」

Claude Code のチャット欄に下を貼って依頼するだけ：

タガル式・自分の教科書スキルを完全に削除してください。
削除するのは次の3つです:
- pip uninstall notebooklm-py
- ~/.claude/skills/study-hub/ フォルダ
- ~/.notebooklm/ フォルダ

「教科書を作ったら、それは公開されちゃう?」

公開されません。Markdownファイルはあなたのパソコンの中だけ、NotebookLM上はあなたのGoogleアカウント内だけ。共有したい時だけ、自分で共有設定をすればOK。

🗺️ 仕組み（5人の連携プレー）

技術用語を全部隠せば「お願いしたら、AIがやってくれた」それだけ。でも裏側を覗いてみたい人のために。

①あなた  →  ②Claude Code  →  ③Macの中の書類  →  ④リモコン  →  ⑤NotebookLM
（指示）    （AI操縦士）    （Markdown）       （notebooklm-py） （Googleのサービス）
登場人物正体役割
①あなた監督テーマを決めるだけ
②Claude Code手足のあるAIファイルを作る・コマンドを撃つ
③ローカルファイルMarkdown書類リサーチ素材・教科書本体
④notebooklm-pyNotebookLMのリモコンターミナルからNotebookLMを操作
⑤Google NotebookLMクラウドの学習サービスあなた専用の学習ハブになる

全体の流れ（時系列で）

[時刻 0:00] あなた「感情の教科書を作って」
[時刻 0:01] Claude Code「観点4つに分けますね。OK?」
[時刻 0:02] あなた「OK」
[時刻 0:03] Claude Codeが分身4人を並列起動
[時刻 0:10] 4分身がそれぞれリサーチ完了
[時刻 0:30] Claude Codeが教科書9章を順次執筆
[時刻 1:30] 教科書完成（11ファイル）
[時刻 1:31] notebooklm-py（リモコン）を呼び出す
[時刻 1:33] NotebookLM上にノートブック誕生
[時刻 1:34] 17ファイルを並列アップロード
[時刻 1:45] マインドマップ即時完成
[時刻 2:00] 音声・FAQ・タイムライン等が順次完成
[完了]      いつでも勉強できる学習ハブ

なぜこの仕組みが成り立つのか

  • 条件1：Claude Codeが「手足を持つAI」だから。普通のAIは会話で終わるが、Claude Codeはコマンドが撃てる。
  • 条件2：notebooklm-pyが「Googleのフタを開けてくれた」から。GoogleがNotebookLMの公式APIを出していなくても、有志（teng-linさん）が公開してくれた。
  • 条件3：NotebookLMが「素材を入れれば全部展開してくれる」から。マインドマップ・音声・FAQまで自動生成。
  • 条件4：すべてが「あなたのアカウント」で動くから。Google OAuthで認証するので、他人のものは触れない。

このメソッドが面白い理由

「AIに使われる」から「AIを使う」への切り替わり点が、ここにあります。

  • 普通の人は、AIに質問して、AIの答えを読む = 「AIに使われる」
  • このメソッドでは、AIに自分の代わりに研究させ、組み立てさせ、学習ハブまで作らせる = 「AIを使う」

しかも、完成した教科書を読んで学ぶプロセスで、自分の言葉が育つ。
だから「AIを使う」状態が、同時に「自分が成長する」状態にもなる。

🔧 セットアップについて

Q. プログラミング知識は必要？
ほぼ不要です。Claude Code Desktop のチャット欄にお願い文をコピペできれば動きます。あとは日本語でAIに話しかけるだけ。

Q. Macじゃないとダメ？
Windows・Linuxでも動きます。Claude Code Desktop が対応しているOSなら使えます。

Q. ターミナル（黒い画面）を別に開きますか？
いいえ。Claude Code Desktop が内部でやってくれるので、別アプリは何も開きません。

Q. Python や pip が無いと言われました
Claude Code に「直して」と書くだけ。AIが brew install python3 など必要なソフトを自動で入れて、続きを進めてくれます。詳細は 🛠 うまくいかなかったとき。

🎓 使い方について

Q. どんなテーマで使える？
何でも。ただし「自分が本気で知りたいこと」が向いています。例：学問（哲学・心理学・歴史）、仕事のテーマ（マーケティング・営業）、個人のテーマ（自己肯定感・人間関係）。

Q. 教科書ができるまでどれくらいかかる？

段階時間
観点分解30秒
並列リサーチ5〜10分
教科書執筆30〜90分
NotebookLM投入2〜5分
成果物生成（マインドマップ・音声等）10〜20分

実時間：合計1〜2時間。 ただし、ほとんどはAIが裏で動いているので、あなたは別のことをしていてOK。

Q. 完成した教科書はどこに残る？
Markdownファイル: あなたのMacの中（永久保存） / NotebookLM: Google上のあなたのアカウント。どちらも消えません。

Q. 自分の作った教科書を売っていい？
もちろんOK。教科書はあなたが作ったものなので、あなたのもの。ただし、このスキル本体（SKILL.md）を再配布する場合はMITライセンスに従ってください。

🤔 哲学について

Q. ChatGPTで質問するのと何が違う？
ChatGPTで質問する → ChatGPTの言葉が返ってくる。
このメソッド → 自分の言葉が育つ。
教科書を作る過程で、自分の中で言語化のプロセスを通るからです。

Q.「目的は教科書を作ることじゃない」ってどういう意味？
教科書は副産物です。本当の目的は、学ぶ過程で自分が変わること。だから、AIに「全部やらせて完成品だけ受け取る」のは推奨しません。必ず章末問題を自分で書き出してください。

🏗 技術アーキテクチャ

配布元・インストール先・CLI・作業フォルダの4層構造です。スキルを改造したい開発者向け。

┌────────────────────────────────────────────────────────────┐
│  GitHub: Tagaru05/tagaru-study-hub  （配布元）             │
│   ├─ skills/study-hub/SKILL.md   ← スキル本体              │
│   ├─ install.sh                  ← 1コマンドセットアップ    │
│   └─ docs/                       ← 使い方ドキュメント      │
└────────────────────────────────────────────────────────────┘
                          │ install.sh が curl で取得
                          ▼
┌────────────────────────────────────────────────────────────┐
│  ローカル ~/.claude/skills/      （インストール先）        │
│   ├─ study-hub/SKILL.md          ← このリポジトリのコピー  │
│   ├─ notebooklm/SKILL.md         ← notebooklm-py 由来      │
│   └─ nohau-library/SKILL.md      ← 別スキル                │
└────────────────────────────────────────────────────────────┘
                          │ スキルが内部で呼ぶ
                          ▼
┌────────────────────────────────────────────────────────────┐
│  notebooklm CLI （pip install notebooklm-py）              │
│   - NotebookLM をコマンドラインから操作                    │
│   - ブラウザログイン状態を ~/.notebooklm に保持            │
└────────────────────────────────────────────────────────────┘
                          │ 成果物の置き場
                          ▼
┌────────────────────────────────────────────────────────────┐
│  ~/Notebook LM操作用/  （日々の作業ワークスペース）        │
│   ├─ 01_sources/   ← NotebookLM 投入前の素材               │
│   ├─ 02_outputs/   ← NotebookLM から取り出した結果         │
│   ├─ 03_drafts/    ← 記事下書き                            │
│   └─ archive/      ← 完了案件の退避                        │
└────────────────────────────────────────────────────────────┘

編集する場所の早見表

やりたいこと編集する場所
/study-hub スキルのプロンプトを変えるこのリポジトリの skills/study-hub/SKILL.md → commit → push
インストール手順を変えるこのリポジトリの install.sh → commit → push
自分の環境のスキルだけ一時的に試す~/.claude/skills/study-hub/SKILL.md を直接編集（リポジトリには反映されない）
notebooklm CLI 自体を更新pip install --upgrade notebooklm-py
日々のリサーチ素材を置く~/Notebook LM操作用/01_sources/

手動セットアップ（自動インストーラーを使わない場合）

pip install notebooklm-py
notebooklm login
notebooklm skill install
mkdir -p ~/.claude/skills/study-hub
curl -fsSL https://raw.githubusercontent.com/Tagaru05/tagaru-study-hub/main/skills/study-hub/SKILL.md \
  -o ~/.claude/skills/study-hub/SKILL.md
notebooklm language set ja
