# 🎓 はじめて使う人のガイド
## tagaru-study-hub を完全初心者向けに解説

> 想定読者：**Claude Code Desktop アプリ**は入れたけど、スキルを入れたことがない人
> 想定時間：読みながらで15分、慣れた人なら3分
> 必要なもの：Claude Code Desktop、Googleアカウント、やる気

---

## 📌 まず大事なこと

このスキルを使うために、**ターミナル（黒い画面）を別途開く必要はありません**。
すべて **Claude Code Desktop のチャット欄にお願い文をコピペするだけ** で完了します。

「えっ、それでいいの?」と思った方、その通りです。
裏側の処理は全部 Claude Code（AI）と install.sh（インストール用スクリプト）がやってくれます。

そして万が一エラーが出ても、**「直してください」とチャットに書くだけ**で AI が自動で解決します。
これが Claude Code Desktop アプリの最大の強みです。

# 🚀 Step 1：Claude Code Desktop を起動して、フォルダを開く

## やること

Claude Code Desktop アプリを起動して、適当なフォルダを「プロジェクト」として開きます。

## 詳しい手順

1. Claude Code Desktop アプリを起動する
2. 「Open Folder」または「Open Project」をクリック
3. 適当なフォルダ（**デスクトップ**でOK）を選んで開く

どこを開いても大丈夫です。スキル本体は後で全プロジェクト共通の場所（`~/.claude/skills/`）に保存されます。

# 📋 Step 2：チャット欄にお願いを貼る

## やること

画面下のチャット欄に、下の文章をコピペして送信します。

```
次のコマンドを実行してください。タガル式・自分の教科書スキルをインストールします。

curl -fsSL https://raw.githubusercontent.com/Tagaru05/tagaru-study-hub/main/install.sh | bash

もしPythonやpipが無いと出たら、それも自動でインストールして続きを進めてください。
```

## 詳しい手順

1. 上のコードブロック（灰色の枠）をコピー
2. Claude Code Desktop のチャット欄にペースト
3. Enter（送信ボタン）を押す
4. Claude Code が **「このコマンドを実行していい?」** と確認してくる
5. 内容を確認して **Yes / Allow / 許可** をクリック

## 何が起きるか

チャットに、こんな実行ログが流れます：

```
🎓 タガル式・自分の教科書 セットアップ
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

▶ Step 1/5: Python の確認...
✅ Python OK

▶ Step 2/5: pip の確認...
✅ pip OK

▶ Step 3/5: notebooklm-py をインストール中...
✅ notebooklm-py インストール完了

▶ Step 4/5: NotebookLM ログイン
   ブラウザが自動で開きます...
```

途中で **ブラウザが自動で開きます**。次のステップで対応します。

## 🆘 エラーが出たら？

そのままチャット欄に下を貼って送信するだけ：

```
上のインストールがうまくいきませんでした。
エラーを見て、原因を直してから、もう一度インストールを最後まで進めてください。
途中で必要なソフト（PythonやHomebrew等）が足りなければ、それも入れてください。
```

これだけで Claude Code Desktop が：

1. エラーログを読む
2. 原因を特定する
3. 解決コマンドを提案して実行する（毎回 Yes をクリックするだけ）
4. インストールを最後まで通す

### よくあるエラーと、Claude が自動でやってくれること

| エラー | Claude が自動でやること |
|---|---|
| `command not found: python3` | Mac: `brew install python3` / Windows: `winget install Python.Python.3` |
| `command not found: pip` | `python3 -m ensurepip --upgrade` を実行 |
| `command not found: brew` | Homebrew 公式インストーラを実行 |
| `Permission denied` | `pip install --user` または `sudo` で再実行 |
| `externally-managed-environment` | `--break-system-packages` 付きで再実行 |
| ブラウザが開かない | `notebooklm login` を単独で再実行して詳細ログから診断 |

# 🔑 Step 3：Googleログイン

## やること

自動で開いたブラウザで、Googleアカウントにログインします。

## 詳しい手順

1. ブラウザが自動で開く
2. 「Sign in with Google」をクリック
3. 普段使っているGoogleアカウントを選ぶ
4. 「許可（Allow）」をクリック
5. 「Authentication successful」と表示されたらブラウザを閉じてOK

## なぜGoogleログインが必要?

NotebookLMがGoogleのサービスだからです。
あなたのアカウントでNotebookLMを操作する権限を、Claude Code に渡すための一回限りの手続きです。

## 不安なポイント

**Q. 個人情報、大丈夫?**
このログインは、あなたのパソコンとGoogleの間だけで完結します。
第三者にあなたのパスワードや情報が渡ることはありません。

**Q. 「アクセス権を要求しています」が怖い**
要求されるのは「NotebookLMへのアクセス」だけ。GmailやDriveには触れません。

- `/study-hub 自己肯定感`
- `/study-hub 人類の歴史`

## 何が起きるか

1. テーマを観点に分解する（あなたに確認します）
2. 4人のAIで並列リサーチ

4. NotebookLMに自動アップロード
5. マインドマップ・音声・FAQを生成

完了したら、ブラウザでNotebookLMを開くと、**あなた専用の学習ハブ**が完成しています。

# 💬 よくある不安への回答

## 「私はプログラミングできないんですが…」
このガイドの通りに、コピペだけで進められます。
中で起きていることを理解する必要はありません。
**料理のレシピを「分子レベルで理解する必要がない」のと同じ**です。

## 「途中で失敗したら全部壊れません?」
**壊れません**。失敗してもパソコンには何の影響もありません。
最悪、何度でもやり直せます。
しかも Claude Code Desktop なら「直して」と書けば AI が自動で復旧します。

## 「変な英語が画面に流れて怖い」
英語のメッセージは、**「いま何をしているか」の実況中継**です。
読まなくて大丈夫。`✅` マークと `🎉` マークだけ見ていればOK。

## 「インストールしたものを後で消したい」
Claude Code Desktop のチャット欄に下を貼って依頼するだけ：

```
タガル式・自分の教科書スキルを完全に削除してください。
削除するのは次の3つです:
- pip uninstall notebooklm-py
- ~/.claude/skills/study-hub/ フォルダ
- ~/.notebooklm/ フォルダ
```

## 「教科書を作ったら、それは公開されちゃう?」
**公開されません**。
- Markdownファイル：あなたのパソコンの中だけ
- NotebookLM上：あなたのGoogleアカウント内だけ
- 共有したい時だけ、自分で共有設定をすればOK

## 「他のテーマでも何度でも使えますか?」
**使えます**。テーマを変えて `/study-hub` を実行すれば、毎回新しい教科書が作られます。

# 🎁 セットアップ完了後にやってほしいこと

セットアップが終わったら、ぜひ：

### 1. テスト用のテーマで小さく試す
いきなり本気テーマじゃなく、**興味あるけど軽め**のテーマで試してください。
例：「コーヒーの淹れ方」「日本の祭り」「猫の心理」

### 2. 完成した教科書を読んでみる
読む過程で「あ、ここわかる」「ここわからない」を意識してください。
**読みながら自分の言葉が育つ**のがこのメソッドの本質です。

### 3. NotebookLMで質問してみる
教科書ができたら、NotebookLMのチャット欄で：
- 「この章を3行でまとめて」
- 「日常生活でどう使える?」
- 「私が一番大事にすべき1点は?」

など、自分なりに質問してみてください。
