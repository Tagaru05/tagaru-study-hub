## 5つの登場人物を1つずつ説明

### ①あなた（人間）

**やること**：
- 日本語で「○○の教科書を作って」と言うだけ

**それ以上は何もしなくていい。** AIが全部やってくれます。

---

### ②Claude Code（クロード・コード）

**正体**：あなたのMacの中で動く「AI操縦士」。

**たとえ話**：
- 普通のAI（ChatGPT等）＝ブラウザの中で会話するだけ
- Claude Code ＝あなたのMacの中に住んで、ファイルを作ったり、コマンドを撃ったり、他のAIを呼んだりできる、**手足のあるAI**

**特技**：
- 日本語の指示を理解する
- 自分の分身（サブエージェント）を作って並列作業できる
- ファイルを作る・読む・書き換える
- ターミナル（黒い画面）でコマンドを実行できる
- 他のAIサービス（NotebookLMなど）と連携できる

**インストール方法**：
- claude.com からダウンロード（無料体験あり）
- 月額プランあり

---

### ③ローカルファイル（Mac内の書類）

**正体**：あなたのパソコンに保存されるテキストファイル。

**たとえ話**：
- AIが書いた「下書きノート」が、あなたのデスクの引き出しに並んでいくイメージ

**今回の例**：
- 約60,000字のリサーチ素材（6ファイル）
- 9章構成の教科書（11ファイル）
- 全部 `.md`（Markdownマークダウン）という拡張子のテキストファイル

**ポイント**：
- これらはあなたのMacの中にあるので、NotebookLMが死んでも消えない
- いつでもVSCode、Word、メモ帳で開ける

---

### ④notebooklm-py（NotebookLMのリモコン）

**正体**：NotebookLMを「コマンド」で操作するための道具。

**たとえ話**：
- 普通の人 → ブラウザでNotebookLMを開いて、マウスでクリック・ドラッグ
- Claude Code → このリモコン（notebooklm-py）を使って、コマンドで一発操作

**どこにあるか**：
- 本体コード：GitHub上で公開（[teng-lin/notebooklm-py](https://github.com/teng-lin/notebooklm-py)）
- 配布場所：PyPI（パイソンの世界の「アプリストア」）
- インストール：ターミナルで `pip install notebooklm-py` 1行

**なぜ存在するか**：
- Googleは公式にNotebookLMのAPI（外部接続口）を公開していない
- でも、ブラウザでログインすれば操作できる
- それを「ブラウザの代わりにコマンドでやる」ように作られたのが、このリモコン
- 開発者の有志（teng-linさん）が世界中の人のために公開してくれている

**仕組みの心臓部**：
- このリモコンがないと、Claude CodeはNotebookLMに触れない
- これがあるおかげで、**「Claude Code × NotebookLM」**という連携が成立する

---

### ⑤Google NotebookLM

**正体**：Googleが提供する、AIを使った学習・整理サービス。

**たとえ話**：
- 自分専用の「家庭教師つき図書館」
- 資料を放り込むと、AIが読み込んで、質問に答えてくれる
- マインドマップ・音声・FAQ・要約まで自動で作ってくれる

**できること**：
- 最大50〜600個のソースを1ノートブックに入れられる（プランによる）
- PDF・YouTube・URL・テキストファイル何でも入る
- ブラウザでアクセス：https://notebooklm.google.com
- スマホアプリもある

**料金**：
- 無料プランあり
- Plus / Pro / Ultra で容量増加

## なぜこの仕組みが成り立つのか（4つの条件）

### 条件1：Claude Codeが「手足を持つAI」だから
普通のAIは会話で終わる。Claude Codeはコマンドが撃てる・ファイルが作れる。

### 条件2：notebooklm-pyが「Googleのフタを開けてくれた」から
GoogleがNotebookLMの公式APIを出していなくても、有志が「ブラウザでできることはコマンドでもできる」と作ってくれた。

### 条件3：NotebookLMが「素材を入れれば全部展開してくれる」から
ソース1つ入れただけで、マインドマップ・音声・FAQまで自動生成。これがなければ手作業で何時間もかかる。

### 条件4：すべてが「あなたのアカウント」で動くから
Google OAuthで認証するので、あなた個人のNotebookLMにアクセスする。他人のものは触れない。安心。

## 誰でも始められる「7ステップ」セットアップ

### Step 1: Claude Codeをインストール
- claude.com にアクセス
- "Download for Mac"（または Windows）をクリック
- 普通のアプリと同じようにインストール

### Step 2: Claude Codeを起動・ログイン
- Anthropicアカウントを作る（無料）
- 無料体験 or 月額プランを選ぶ

### Step 3: ターミナルを開く
- Mac: アプリケーション → ユーティリティ → ターミナル
- Windows: スタート → コマンドプロンプト

### Step 4: notebooklm-pyをインストール
ターミナルにこの1行を貼る：
```
pip install notebooklm-py
```

### Step 5: NotebookLMにログイン（リモコンの初期設定）
```
notebooklm login
```
ブラウザが開いて、Googleアカウントでログイン。

### Step 6: スキルをインストール（オプション）
```
notebooklm skill install
```
これでClaude Codeが「NotebookLMの使い方」を覚える。

### Step 7: Claude Codeに話しかける
```
○○の教科書を作って
```

これで終わり。**初期設定は5分・初心者でも可能**。

## あなたが知らなくていいこと（裏側）

技術好きな人向けの裏側情報。**知らなくても全く問題なく使えます。**

- pip = Pythonライブラリのインストーラー
- PyPI = Pythonライブラリの配布場所
- CLI = コマンドラインインターフェース（黒い画面の操作）
- OAuth = ログイン認証の仕組み
- API = サービス同士の接続口
- リバースエンジニアリング = 公式APIがないものを解析して使えるようにする技術
- Markdown = 簡単な記法で書ける文書フォーマット
- サブエージェント = Claude Codeが起動する子AI

## よくある質問

### Q. お金はどれくらいかかる？
- Claude Code: 月額20ドル〜（無料体験あり）
- NotebookLM: 無料プランあり、有料プランは月数千円〜
- notebooklm-py: 完全無料（GitHubで公開）

### Q. プログラミング知識は必要？
**ほぼ不要**。インストール時に1〜2行のコマンドをコピペするだけ。あとは日本語でAIに話しかけるだけ。

### Q. 失敗したら？
失敗してもファイルは残るので、いつでもやり直し可能。AIが「ここでエラーが出ました」と教えてくれる。

### Q. 自分のPCに何が残る？
- Markdownファイル（あなたが作った教科書とリサーチ素材）
- NotebookLMの中身はクラウド（あなたのGoogleアカウント内）

### Q. 他の人に見せられる？
NotebookLMには共有機能あり。URLを渡せばOK。教科書ファイルもMarkdownなのでメール・チャットで送れる。

## このメソッドが面白い理由

「**AIに使われる**」から「**AIを使う**」への切り替わり点が、ここにある。

普通の人は、AIに質問して、AIの答えを読む。
これは「**AIに使われる**」状態。

このメソッドでは、AIに自分の代わりに**研究させ、組み立てさせ、学習ハブまで作らせる**。
人間は「テーマを決めて、過程を見守る」だけ。

しかも、**完成した教科書を読んで学ぶプロセスで、自分の言葉が育つ**。
だから「AIを使う」状態が、同時に「自分が成長する」状態にもなる。

**これが、タガル式が単なる効率化ツールじゃない理由です。**
