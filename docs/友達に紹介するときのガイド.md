# 👥 友達に紹介するときのガイド

> 想定読者：「この学習ハブ、友達にも教えたい」と思った人
> ITが苦手な相手にも、**コピペ3回**で使ってもらえる手順をまとめました。

---

## 🎯 結論：相手に送るのはこの3行だけ

LINEやメールに**そのままコピペ**して送ってください。

```
学びたいテーマで「自分専用の教科書」を一晩で作れるツールを共有します。

▼ 使い方（3分でセットアップ）
https://github.com/Tagaru05/tagaru-study-hub

リンク先のREADMEに「5分でセットアップ」と書いてあるところを上から順にやればOK。
分からなければ「はじめて使う人のガイド」（リンク先内）も読んでみて。

困ったら聞いてね！
```

これだけで、相手は自分で読みながらセットアップできます。

## 🛡️ 相手にトラブルが起きたとき

相手から「○○できない」と言われたら、まずは以下を案内：

| 相手の状態 | あなたが案内するもの |
|---|---|
| そもそも何をしていいか分からない | [はじめて使う人のガイド](はじめて使う人のガイド.md) |
| よくある質問・不安 | [FAQ](FAQ.md) |
| 仕組みを詳しく知りたい | [仕組み解説](仕組み解説.md) |
| 「Python が無い」「pip が無い」と言われた | はじめて使う人のガイドの「トラブルシューティング」セクション |
| 「Claude Code が分からない」 | https://claude.com の Sign up からアカウント作成 → Download |

それでも解決しなければ、**スクリーンショットを送ってもらう**のが最速です。
ターミナルの画面を画像で送ってもらえば、どこで止まっているか一発で分かります。

## 💡 相手のリテラシー別・声かけのコツ

### 🟢 普段からパソコンを触る人
- リンクだけ送ればOK
- 「README見て」で十分

### 🟡 メール・ブラウザは使えるけど Terminal は知らない人
- LINEテンプレ（上記1️⃣）を送る
- 「ターミナルって黒い画面のこと」と一言添える
- 「分からなくなったら『はじめて使う人のガイド』読んでみて」と保険を貼る

### 🔴 「私プログラミングできないんで…」と最初から構える人
- 一言：**「コピペするだけだから大丈夫」**
- 「料理のレシピを分子レベルで理解する必要がないのと同じだよ」と伝える
- できれば**最初の1回は画面共有で一緒にやってあげる**のが一番確実
- セットアップさえ越えれば、あとは日本語で話しかけるだけ

## 🔄 配布元（このリポジトリ）の更新

スキルを改善してアップデートを配布したくなったら：
