# /study-hub ― 自分の教科書を作るスキル

このスキルは、りょうさん（タガル）の発見した**「タガル式・自分の教科書」メソッド**を、誰でも使える形に落とし込んだものです。

## 目的

**「学びを言語化するプロセスで、自分が言葉を持つ」こと。**

教科書を作るのは手段。
目的は、学ぶ過程で自分の中に **「自分の言葉」** が育つこと。

---

## 起動条件（明示・意図判定）

### 明示的起動
- 「/study-hub [テーマ]」
- 「[テーマ]の教科書を作って」
- 「[テーマ]について自分の言葉で語れるようになりたい」
- 「[テーマ]の学習ハブを作って」

### 意図判定で起動
- 「○○について体系的に学びたい」
- 「○○のリサーチをして、NotebookLMに入れたい」
- 「○○を出版本のテーマにしたい」

---

## 必須前提

1. `notebooklm` CLI インストール済み・認証済み
2. NotebookLM 言語設定が日本語（`notebooklm language set ja`）
3. 作業フォルダがある（デフォルト: `~/Desktop/総合フォルダー/[テーマ]リサーチ/`）

---

## 標準ワークフロー（7ステップ）

### STEP 0：テーマの受領と確認

ユーザーから「○○について教科書を作りたい」と受けたら、まず確認：

1. **テーマの明確化**
   - 「○○とは何か」を語れるようになりたい、で合っているか
   - 範囲は広め／狭めどちらか

2. **目的の確認**
   - 出版本の土台？ 発信用？ 自分の整理？
   - 想定読者は自分だけ／クライアントも／世間一般？

3. **必要な深さ**
   - 軽め（半日完成）／標準（1晩）／徹底（数日）

---

### STEP 1：観点の分解（3〜5本）

テーマを多角的に切る観点を提案する。

#### 観点の標準パターン

| 観点 | 切り口 |
|---|---|
| 科学班 | 心理学・脳科学・神経科学・統計データ |
| 哲学班 | 思想史・古典・現代哲学・東洋思想 |
| 言語化班 | テーマと言葉の関係・言語化の科学 |
| 実践班 | 具体的なやり方・コーチング・現場応用 |
| AI班 | AI時代における新しい論点・最新研究 |
| 歴史班 | 時系列での変遷 |
| 日本特有班 | 日本人の価値観・文化との接続 |

テーマに応じて3〜5本選ぶ。

**ユーザーに確認**：「この観点で進めてOK？ 追加・変更ある？」

---

### STEP 2：並列リサーチ実行

各観点を**サブエージェントで並列調査**する。

```
Agent × 観点数 並列実行：
- description: "○○の[観点]リサーチ"
- subagent_type: "general-purpose"
- prompt: 観点に応じたリサーチ指示
- run_in_background: true
```

**重要**：サブエージェントには「**ファイル書き込みはせず、本文をテキストでチャットに返す**」と明示する（権限拒否回避）。

各エージェントの出力：
- 約2,000〜3,000字
- 構成：TL;DR → 主要論点 → 「○○とは○○である」3パターン → 引用元

---

### STEP 3：統合サマリー作成

並列リサーチの結果を受け取ったら、メインエージェント（ウィザーモン）が統合する。

統合サマリーの構成：
- TL;DR（200字）
- 3つの核（要点を3つに圧縮）
- 4観点の比較表
- 「自分の言葉で語る」フレーズ5本
- 即使える「問いかけテンプレ」10個
- 押さえておきたい人物リスト
- 次の深掘り候補

ファイル名：`00_りょうさん用サマリー.md`

---

### STEP 4：教科書化（章立て + ワーク + 章末問題）

リサーチ素材を元に、9章構成の教科書を組み立てる。

#### 標準章立て（9章）

| 章 | タイトル | 内容 |
|---|---|---|
| 序章 | なぜ○○を学ぶのか | 動機と出発点 |
| 第1章 | ○○とは何か | 定義の歴史と現代 |
| 第2章 | ○○の正体 | 仕組み・構造 |
| 第3章 | ○○の地図 | 種類・分類 |
| 第4章 | ○○と言葉 | 言語化との関係 |
| 第5章 | ○○を読む技術 | 観察・命名 |
| 第6章 | ○○と付き合う技術 | 実践アプローチ |
| 第7章 | ○○とAI | AI時代の論点 |
| 終章 | 自分の言葉で語る | 統合と最終ワーク |
| 付録 | 索引と用語集 | 五十音順検索 |

**各章の必須パーツ**：
- 🎯 この章のゴール
- 📖 本文
- 🎬 日常のシーン
- ✍️ ワーク
- 📝 章末問題

ファイル：`教科書/00_目次.md` 〜 `教科書/索引と用語集.md`

---

### STEP 5：NotebookLMへ投入

教科書とリサーチ素材を、NotebookLMノートブックにまとめてアップロード。

```bash
# ノートブック作成
notebooklm create "○○の教科書 ― 自分の言葉で語るための学習ハブ" --json

# 全ファイルを並列投入
notebooklm source add [各ファイル] --notebook [id] --json

# 処理完了確認
notebooklm source list --notebook [id] --json
```

---

### STEP 6：成果物のフル生成

NotebookLMの全機能を起動：

| 成果物 | コマンド | 用途 |
|---|---|---|
| マインドマップ | `generate mind-map` | 全体俯瞰 |
| 学習ガイド | `generate report --format study-guide` | 体系学習 |
| FAQ | `generate report --format custom`（30問指定） | 逆引き |
| ブリーフィングドキュメント | `generate report --format briefing-doc` | 一気読み |
| タイムライン | `generate report --format custom`（年表指定） | 歴史把握 |
| 音声概要 | `generate audio` | 移動時学習 |

---

### STEP 7：使い方ガイドの提示

ユーザーに「これからどう使うか」を渡す：

#### 朝のフロー
- マインドマップで全体俯瞰（30秒）

#### 昼のフロー
- FAQから1問選んで自分なりに答える

#### 夜のフロー
- チャットで深掘り → 良い回答は「Save as Note」

#### 移動時間
- 音声概要を聴く

## 守るべき軸（レオモン承認）

このスキルを使う時、必ず守ること：

1. **「メソッドを売る」のではなく「道具を渡す」**
   - 目的は教科書じゃない。自分が言葉を持つこと

2. **必ず実例（自分の経験）と接続**
   - 「私はこのテーマで何が変わったか」を必ず書き残す

3. **「言語化」軸を死守**
   - 「学習法」「勉強法」「効率化」の語は使わない
   - 「自分の言葉を見つける道具」と表現する

---

## 戦略上の注意（ピッコロモン承認）

このメソッドの発信について：

1. **教科書本体は公開しすぎない**
   - フリー記事ではプロセスと変化を語る
   - 教科書本体は出版本に温存

2. **実例の数で勝負**
   - 最低3つの異なるテーマで実例を作る

3. **メソッド名「タガル式・自分の教科書」を先名乗り**
   - 6ヶ月以内に発信で口にする

## トラブルシューティング

| 問題 | 対処 |
|---|---|
| サブエージェントがファイル書き込み拒否される | 「本文をテキストで返す」明示で再実行 |
| NotebookLM source add が時間かかる | `source wait` で待機 |
| マインドマップ以外の生成が失敗 | 5-10分後に再試行（Google側のレート制限） |
| 教科書の章立てがしっくり来ない | テーマに応じて章名・順序を調整 |
